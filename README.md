# 🎲 DICEボドゲリスト

所属しているボードゲームサークルの部室にあるボードゲームを整理するために作った管理 Web アプリです。部員が誰でもブラウザから、ゲームの検索・プレイ済み記録・貸出記録を行えます。

**🔗 デモ:** https://dice-board-game-list.vercel.app/
**📦 リポジトリ:** https://github.com/ryogaaoyama-glitch/DICE_BoardGame_List

## できること

主な機能は検索・プレイ済み記録・貸出記録の3つです。

- **検索・絞り込み・並び替え**：ゲーム名/棚番での日本語あいまい検索(ローマ字↔ひらがな↔カタカナを相互変換してマッチ)、言語・重さ・人数・タグでの複合フィルタ、棚番/名前/時間でのソート
- **プレイ履歴管理**：プレイ済み/未プレイをユーザーごとに記録(メールアドレスでログイン、部員が共有して使っても履歴が混ざらない)
- **貸出/返却トラッキング**：誰に貸しているかを記録し、貸出中/在庫ありでタブ切り替えして確認

このほか、棚番・タグ・プレイ人数・難易度(軽/中/重)を一覧できる在庫管理や、localStorage による5分キャッシュ(体感速度の確保とバックグラウンド更新)、プレイ済みトグルなどの楽観的UI更新+失敗時ロールバックも備えています。

ゲームの詳細情報(よみがな・画像URL・備考・タグなど)の追加・編集は、サイト内の編集モーダル、またはスプレッドシートを直接編集することでのみ行います。コード側を触る必要はありません。

## 技術構成

| レイヤー | 使用技術 |
|---|---|
| フロントエンド | Vanilla JS / HTML / CSS(単一 `index.html`、フレームワーク不使用) |
| ホスティング | Vercel |
| APIプロキシ | Vercel Serverless Function(`/api/proxy`)が Google Apps Script Web App にリクエストを中継(CORS回避) |
| バックエンド | Google Apps Script(`doGet` ベースのAPI。JSONPレスポンス対応でスマホブラウザのセキュリティ制限も回避) |
| データストア | Google スプレッドシート(「ボドゲ一覧」シートが本体データ、「プレイ済み」シートがユーザー別プレイ履歴) |
| 外部API連携 | BoardGameGeek XML API(画像URL未設定ゲームの一括補完。Apps Scriptエディタから手動実行するメンテナンス用スクリプトで、アプリの通常機能ではない) |
| 認証 | メールアドレスによる簡易ログイン(ユーザーごとにプレイ履歴を分離) |

### アーキテクチャ

```
ブラウザ (index.html)
   │  fetch('/api/proxy?action=...')
   ▼
Vercel Serverless Function
   │  中継(CORS回避)
   ▼
Google Apps Script Web App (doGet)
   │  読み書き
   ▼
Google スプレッドシート
   ├─ 「ボドゲ一覧」シート … 棚番/名前/重さ/時間/人数/言語/備考/貸出/よみがな/画像/タグ
   └─ 「プレイ済み」シート … rowIndexごとに、ユーザーのメールを列ヘッダーとして動的追加し ✅ を記録
```

無料で使える構成(Vercel + GAS + スプレッドシート)だけでバックエンド・DB・ホスティングを揃え、外部サーバーの契約なしでフルスタックのアプリを運用できるようにしています。

### バックエンドAPI(Google Apps Script)

`doGet` で `action` パラメータを見て以下を振り分け。GASの実行はコールドスタートが遅いため、5分おきに `keepWarm` トリガーを回して待ち時間を抑えています。

| action | 処理内容 |
|---|---|
| `getGames` | スプレッドシートからゲーム一覧を取得(空行除外、rowIndexを主キーとして付与) |
| `lendGame` | 貸出中判定→貸出者名をH列に記録(二重貸出を防止) |
| `returnGame` | H列をクリアして返却を記録 |
| `getPlayed` | 「プレイ済み」シートから、指定メールアドレスの列で✅が付いた rowIndex 一覧を返す |
| `togglePlayed` | 該当ユーザーの列が無ければ動的に追加し、✅の有無をトグル |
| `updateGame` | 棚番・よみがな・画像URL・備考・タグを該当行に書き込み |

「プレイ済み」シートはユーザーごとに列を動的生成する設計にすることで、何人が使っても後からユーザーを追加でき、スキーマ変更なしでスケールするようにしています。GASコード本体は [`gas/Code.gs`](./gas/Code.gs) を参照してください(リポジトリに追加する場合は `gas/Code.gs` として配置してください)。

## 開発の背景

部室のボードゲームが増えて「棚のどこにあるか」「誰かに貸したままか」「まだ遊んでいないか」が部員間で分からなくなったことがきっかけです。スプレッドシート単体だと部室外から見づらく貸出記録も手間だったため、スマホからでも使えるUIと貸出/返却フローをワンタップで完結できるように設計しました。ゲームの詳細追加や画像設定はサイトの編集モーダルかスプレッドシートの直接編集で完結させ、コードをいじる必要がないようにしています。画像は1件ずつ手動で貼るのが面倒な場合に備えて、BGGから自動取得するメンテナンス用スクリプトも別途用意しています。

## ローカルでの実行

静的ファイルのみで動作するため、`index.html` をそのままブラウザで開くか、`api/proxy` を経由させる場合は Vercel CLI でデプロイしてください。

```bash
vercel dev
```

GAS側は本リポジトリ直下の `Code.gs`(リポジトリに入れる際は `gas/Code.gs` に配置)をスプレッドシートに紐づいた Apps Script プロジェクトにコピーし、ウェブアプリとしてデプロイしてください。
